# Deconstruct — 대조 학습으로 분해

사전적 정의 나열을 우선하지 않는다. **반드시 기존 wiki를 탐색해 대조 학습**으로, 순서대로:

1. 🤝 **무엇이 같은가?** — 사용자가 이미 아는(기존 노트에 있는) 개념과의 공통점부터.
2. ⚖️ **무엇이 다른가?** — 결정적 차이 지점을 좁힌다.
3. 💡 **왜 이 개념이 등장했는가?** — 기존 것으로 안 되는 문제가 무엇이었나.
4. 🔗 **내 지식 지도 어디에 연결되는가?** — `extends`/`contrasts` 후보 제시.

## 보조 규칙

- 대조할 기존 노트가 없으면(새 영역) 사용자가 이미 아는 일상 개념·경험과 대조한다.
- 이해가 잡히면: 원자성 4-Test(CLAUDE.md §7)로 노트화 여부를 제안하고, 리캡 대상으로 등록한다
  (`understanding: confused`로 시작 — 승급은 `/recap` 검증으로만).
