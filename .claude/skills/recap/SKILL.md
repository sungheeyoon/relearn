# Recap — 간격 반복 + Feynman 인출

CLAUDE.md §6(status/understanding 정의 · 7-게이트)과 §11(understanding 임의 변경 금지)을 전제한다.
장기기억은 *간격 반복 + 인출 연습(Feynman)*으로 만든다.

## 대상 선정

- 특정 노트 지정이 없으면 raw 노트의 `last_recap`을 **직접 스캔**해(AI는 Dataview 실행 불가) 오래된 순으로 고른다.
- `last_recap`이 빈 노트 = **인출 미검증** → 최우선.
- 사람용 자동 뷰는 [[recap-dashboard]](Obsidian용).

## 간격 스케줄 — 한 세션에 몰아 올리지 않는다

- `confused` → **2~3일** 안에 재학습 (아직 안 굳음, 우선 재이해)
- `clicked` → **3일 → 7일 → 14일**로 간격 확장
- `teachable` → **30일**마다 유지보수 복습

## 절차

1. `## 🧠 My Process & Understanding`를 **제외한** 본문을 우선 파싱한다.
2. 핵심 개념을 1~2줄로 환기시킨다.
3. **Feynman 기반 질문 1개**를 생성해 사용자가 스스로 설명하게 유도한다.
4. 답변을 평가한다.
5. `understanding` 변경을 **제안**한다 → 사용자 동의 후 갱신하고 `last_recap`을 오늘 날짜로 기록
   (다음 기한이 자동으로 뒤로 밀린다).

## 승급 규칙

- "이해했다 / 이해한 것 같다"는 자기보고만으로 올리지 않는다. 반드시 설명·적용·예시·반례 질문으로
  검증한 뒤 판정한다. 자기보고 발화는 **리캡 큐 등록 신호**로 받아 inbox에 리마인더를 남긴다.
- `teachable` 승격은 7-게이트(§6) 중 **미검증 항목을 콕 집어** 질문한다:
  ① 정의 ② 왜 필요 ③ 무슨 문제 해결 ④ 언제 사용 ⑤ 반례 ⑥ 직접 예제 ⑦ 남에게 설명.
- 반복해서 막히면 대조 학습(`/deconstruct`)으로 전환을 제안한다.

## 마무리

- 커밋 제안: `recap(노트명): 요약` + 트레일러 `Understanding: confused → clicked`(§12).
- 행동층(§14) 사용 시, 세션 학습 활동이 끝나면 log.md 기록은 `/today` 규칙을 따른다.
