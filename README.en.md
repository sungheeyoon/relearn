# 🧠 Relearn

[한국어](./README.md) · **[Introduction site →](https://getrelearn.vercel.app)**

> A personal knowledge system where you collect what you study, and AI organizes, links,
> and — most importantly — **cuts the cost of re-learning what you forgot**.

Built on [Andrej Karpathy's "LLM Wiki" pattern](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f),
extended for **re-learning**. View it in Obsidian, let Claude Code maintain it, keep history in Git.

```
Raw Sources → AI Ingest → Atomic Notes → Maps(MOC) → Recap Loop → Long-term Knowledge
```

## Why

1. **Reduce re-learning cost, not preserve memory** — Humans forget by design. The goal is not
   "never forget" but **recover fast** after forgetting. Your misconceptions and how you actually
   came to understand things are recorded as recovery hints.
2. **Full-context wiki, not RAG** — At personal-knowledge scale (~50–100k tokens), an AI that
   reads the whole wiki beats vector search: 100% retrieval reliability, zero infrastructure.
3. **AI as librarian, human as judge** — The real labor of a knowledge base is not thinking but
   **bookkeeping** (cross-referencing, filing, linting). An AI never gets bored and never forgets,
   so it does that part; you curate sources and ask questions.

Two additions on top of the original pattern:

- **Recap Loop** — Spaced repetition + Feynman-style retrieval practice. The AI picks overdue notes,
  asks you to explain, and only promotes your understanding level (`confused → clicked → teachable`)
  after verifying your answer. Self-reporting doesn't count.
- **Behavior layer (optional)** — A minimal habit system (floor / standard / ceiling + ratchet)
  that manufactures "study days" in the first place. Delete it if you don't need it.

## Getting started

1. Hit **[Use this template]** to create your own repo (private recommended — this is where your
   personal learning history accumulates)
2. Clone, then open the folder in Obsidian via **"Open folder as vault"** → enable the **Dataview**
   community plugin
3. Open the folder in a terminal and run **Claude Code** — the rules in `CLAUDE.md` load automatically
4. Drop anything you studied into `sources/` (or a one-liner into `inbox/`) → say **"ingest this"**

Two example notes ship with the template (`llm-wiki-pattern`, `zettelkasten`). They document the
system itself, so using them as your first recap material teaches you the system while you use it.

## How it feels

Each workflow is a separate **Claude Code skill** (`.claude/skills/`) — invoke them directly as
`/ingest`, `/recap`, `/deconstruct`, `/lint`, `/today`, or just say it in natural language as below.
Skills load only when invoked, keeping the everyday session context light.

**Ingest — sources to notes**
> Me: I put today's reading in `sources/`. Ingest it.
> AI: Read it, created 2 atomic notes. `closure` was merged into an existing note as a section
> (failed the atomicity 4-test), linked via `topics: [javascript]`. Proposed commit:
> `ingest(closure): closure & scope study notes`

**Recap — the core loop**
> Me: Let's recap.
> AI: Oldest `last_recap` is [[zettelkasten]] (12 days). Question: why did "links need context"
> matter in Zettelkasten? How is it different from Wikipedia links?
> Me: (explains in my own words)
> AI: Core is right, but you missed "linking at the level of individual thoughts". I propose
> keeping `clicked` — if you agree I'll update `last_recap`.

**Query — answers grounded in *your* notes**
> Me: Based on my notes, what was the decisive difference between Zettelkasten and the LLM Wiki?
> AI: According to [[zettelkasten]] and [[llm-wiki-pattern]], it's *who forges the links* — … (evidence as links)

**Lint — hygiene check**
> Me: Lint the vault.
> AI: 1 orphan note, 2 broken links, 1 source missing `origin`. Here are the fixes — applied on approval.

## Structure

| Path | What |
|------|------|
| `sources/` | Raw study material (human adds, AI is **read-only**) |
| `notes/concepts/` | Concepts, theories, methods, patterns — objective summaries the AI maintains |
| `notes/entities/` | People, tools, organizations, papers |
| `notes/maps/` | MOC (Map of Contents) hubs — lists auto-generated by Dataview |
| `inbox/` | Quick capture (emptied on ingest) |
| `schema/` | Note template · behavior-layer rules |
| `00-index.md` | Home dashboard |
| `CLAUDE.md` | **The heart of the system** — schema & permission model (loaded every session) |
| `.claude/skills/` | Workflow skills (`/ingest` `/recap` `/deconstruct` `/lint` `/today`) — loaded on demand |

## Design principles (summary)

- **Sustainability over perfection** — under 30 notes, never grow the structure. Just record and recap.
- **Objective vs subjective, separated** — the AI writes source-grounded summaries; each note's
  `🧠 My Process & Understanding` section belongs to the user and the AI never touches it uninvited.
- **Contradictions are never deleted** — they get linked via `contradicts` and kept as open questions.
- **Provenance everywhere** — every fact must trace to `sources` or an inline `(source: [[..]])`.
- **Permission model** — additive changes (links, frontmatter, new notes) are automatic; destructive
  ones (delete, merge, rename, push) require approval. **No auto-push, ever.**

The full ruleset lives in [`CLAUDE.md`](./CLAUDE.md) (currently in Korean). The file *is* the system —
rewrite any rule you disagree with.

## Optional module: the daily system

The knowledge layer only works on days you study. [`schema/daily-system.md`](./schema/daily-system.md)
is the behavior layer that manufactures those days — a daily minimum (floor), the real target (standard),
an overheat cut-off (ceiling), and a ratchet that adjusts difficulty only from two weeks of data.
The AI acts as a coach but **never judges you.** Don't need it? Delete `CLAUDE.md` §14,
`schema/daily-system.md`, and `log.md`.
